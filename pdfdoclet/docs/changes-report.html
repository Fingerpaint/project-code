Release 1.0.2 - 10-12-2005

TypeChangesBy
Custom tag names are now handled case-sensitive (as they should be!). Fixes 1327157. marcelschoen
Multiple inheritance of interfaces was not handled correctly. The "extends ..." text contained only one of the inherited interfaces. Fixes 1313937. marcelschoen
The doclet generated a 0-length PDF file, if an invalid font was specified. An appropriate error message was printed for the missing font, but then the doclet continued and failed later with a NullPointerException, leaving a corrupt file behind. Now the doclet cancels processing if a font path is invalid and deletes the PDF file. Fixes 1375788. marcelschoen
Field and methods within a class bookmark entry are now sorted alphabetically. Fixes 1293618. marcelschoen

Release 1.0.1 - 11-09-2005

TypeChangesBy
Added support for more HTML tags: STRONG and Definition lists (DL / DT / DD). Fixes 1256563. marcelschoen
Made PRE (sections of preformatted code) nicer and better distinguishable from the rest of the page, by putting them into a box with a black border and light-gray background color. Fixes 1256562. marcelschoen
Added support for specifying several files from which to import pages into the overview, using numbered property names, such as "overview.pdf.file.1", "overview.pdf.file.2" etc. marcelschoen
The spacing before some Hx-tags (like H1, H2..) was smaller than the spacing after the tag. This lead to the impression th at the heading belonged to the previous parapgraph instead of the next. Fixes 1256571. marcelschoen
Font path names were not used relative to the specified "workdir" property value. Fixes 1256540. marcelschoen
Classes within a package in the bookmarks were not sorted alphabetically. Fixes 1256542. marcelschoen
The CENTER tag did not work anymore. The reason was a JTidy setting which caused CENTER tags to be replaced with DIV tags. Fixes 1283004. marcelschoen

Release 1.0.0 - 03-07-2005

TypeChangesBy
Added support for importing pages from PDF files in appendices, before the overview and for the title page. marcelschoen
Added support for including custom HTML files as appendices before the index. marcelschoen
Massively improved bookmarking code in order to support the "-group" parameter and allow for a more detailled bookmark tree per class entry (links to constructors, fields and methods). marcelschoen
Added support for anchors ( < a name="..." > ) marcelschoen
Added support for custom bookmark entries, using special anchors ( < a name="pdfbookmark" label="...." > ). This is especially useful if an overview is used which spans several printed pages. Together with the HTML comment used to trigger page breaks, this feature helps to improve the structure and usability of the generated PDF. marcelschoen
Configuration switch "inherited.summary.table" added for enabling / disabling printing of summary tables for inherited methods and fields. The property also allows to print only summary tables for elements inherited within the scope of the API, but not of met hods, fields etc. inherited from standard classes such as " java.lang.Object". marcelschoen
Bulleted lists are now displayed with an actual bullet symbol instead of a dash character. marcelschoen
Added some leading space for regions of PREformatted text, just as in HTML pages. marcelschoen
Handling of "author", "version" and "since" parameters / properties didn't work correctly anymore. This has been fixed, so that printing of those tags can be suppressed again. marcelschoen
Minor layout / spacing improvements. marcelschoen
Added parameters for overriding labels of bookmark entries for fields, constructor and methods outlines. marcelschoen

Release 0.9.0 - 18-04-2005

TypeChangesBy
Added support for custom filtering. It is now possible to create PDF output only for certain classes / methods / members, based on custom tags. See the "Usage" page of the HTML documentation for details. The main reason for this feature is to allow to create extracts of an API based, for example, on project or developer names etc. marcelschoen
Added support for printing any custom tags (such as "@todo"). A custom tag like "@todo" will be ignored, though, as long as there is no label defined for it in the configuration. A label definition for a custom "todo" tag could be: labels.tag.todo=To Do: marcelschoen
"workdir" was processed as a configuration property only, but not as a commandline argument. Fixes 1183868. marcelschoen
A-tags (links) without a "href" field caused a NullPointerException in the TagA class. Fixes 1186422. marcelschoen
Added printing of fields and methods inherited from any interface in the "... inherited" tables. marcelschoen
Cleaned up the "test-pdf" example sources. marcelschoen
Added note about custom tag to FAQ. marcelschoen
Some refactoring (extracted some large methods from the main class "PDFDoclet" which has now, at long last, achieved a reasonable size. marcelschoen

Release 0.8.2 - 06-03-2005

TypeChangesBy
Resolving the "inheritDoc"-inline-tag was implemented in a way which could not deal with recursive inheritance over several levels of classes (subsubclass inherits do c from subclass which inherits doc from subclass etc.). The fix implements a proper, recursive resolution. The test-pdf example has been exteded to test this (classes OtherClass, SubClass and TestClass). Fixes 1156553. marcelschoen
Included Log4j for debug/info logging. If it is called with the "-debug" flag, it prints plenty of debug output to the logfile "pdfdoclet.log" and some info log to the console. marcelschoen

Release 0.8.1 - 09-02-2005

TypeChangesBy
The annoying missing-blanks-problem has finally gone away. The secret lied in using JTidy differently (and maybe also in using a more current version of JTidy). I won't go in much detail here, but I'll just say that it should no longer be necessary to encode blanks or insert them before an ending tag just to get them out in the PDF. Fixes 849061. marcelschoen
ClassCastException caused by open list tags has been solved by the fact that JTidy now actually tidies up the HTML. Fixes 1058391. marcelschoen
Implemented better error handling for IOException which occurs if the PDF file cannot be opened for writing. marcelschoen
Shut up JTidy. This was necessary because currently i'm using a snapshot release of JTidy which creates lots of debug output. marcelschoen

Release 0.8.0 - 14-01-2005

TypeChangesBy
Java 1.4 is now required. This is due to the fact that PDFDoclet now uses some methods of the doclet API available only from 1.4 on. Sorry, if that's a problem to anyone. However, I think that nowadays Java 1.4 should be available on almost any platform. If not, maybe it's possible to run it with Java 1.2/1.3 and just include the "tools.jar" from 1.4 in the classpath. I've no idea if that's possible, just an idea that came to my mind. You may try that if you're really desperate... marcelschoen
Switched to Maven as the build tool. marcelschoen
Included the most current snapshot release of JTidy. marcelschoen
Implemented creation of single Jar file which contains all dependencies to make javadoc execution easier. marcelschoen
Constant values are now printed in field summaries. marcelschoen
Implemented printing of inherited methods and fields in summary of interfaces (which extend another interface). marcelschoen
Support for {@inheritDoc} inline tag Implemented for both method doc blocks and tags Fixes 913036. marcelschoen
New "font.." parameters (see README) allow to use other fonts by embedding TrueType fonts (ttf). marcelschoen
NullPointerException fixed which occurred if an image tag had no "src" attribute (for example due to a typo, like "srx=") marcelschoen
Overview pages had no header and footer and therefore no page numbering. This has been fixed. Fixes 972464. marcelschoen
Until now, in method signatures, only for the return type a link was created, but not for the parameters. Now, the parameters are links too (depending on their type, of course). Fixes 894692. marcelschoen
Implemented correct support for @see tags and inline links like {@link ..}, inluding correct linking to the actual member (method, variable), not just the containing class. Fixes 931060. marcelschoen
Change the boomark entry of the title In the outline frame (the bookmarks), the labels were somewhat inconsistent ("Title Page", but "Overview" and "Index", not "Overview Page" etc.). The defaults are now consistent (without "page"). The labels can also be customized with 3 new configuration properties: "label.bookmarks.index", "label.bookmarks.title" and "label.bookmarks.overview" Fixes 894560. marcelschoen
Changed IConstants.LB_EXCEP_TAG value to "Throws:". The label "Exceptions:" was replaced by "Throws:" Fixes 894449. marcelschoen
Allow all options to be set in the config file or as params. All configuration values may now be specified either in a property file or as commandline arguments. There is a new section in the README file which explains all configuration properties. Fixes 894506. marcelschoen
Several still hard-coded parameter names or values replaced Fixes 894488,894544,894478. marcelschoen
Add an 'Overview' entry into the bookmarks The "overview.html" page now also has its own entry in the bookmarks, just behind the "Title Page" entry. Fixes 894562. marcelschoen
When the first sentence contained package names like "java.util...", only the part of the text up to the first dot "." was taken as the first sentence (used in the summary tables). Fixes 895216. marcelschoen
NullPointerException for unvalued attribute HTML-tag attributes without values, such as NOWRAP in a TABLE, created a NullPointerException. This has been fixed. Fixes 895261. marcelschoen
Parameters in bookmarks stopping after the first one Methods in bookmarks now contain all parameters, but only their types (to keep the bookmarks short), like this: doSomething(String,Properties) Fixes 894673. marcelschoen
Error messages need some context info In debug mode, the member (method, variable..) currently being processed is now also printed. This makes it easier to locate HTML problems mentioned by JTidy warnings. Fixes 895226. marcelschoen
Some mixed case text has been fixed for better consistency. Fixes 894708. marcelschoen

Release 0.6.0 - 06-11-2005

TypeChangesBy
Added option for custom order of packages in the bookmarks frame (new parameter " sort " , as commandline argument or as configuration parameter - see the " config_test.properties " file for an example). marcelschoen
Added support for the {@docRoot} tag introduced in JDK 1.3. It's now being replaced with the working directory value, which is just " . " if unspecified. Fixes 800912. marcelschoen
Added option for specifying packages whose classes should not be fully qualified (new parameter " dontspec " as commandline argument or as configuration property). Example: By setting " dontspec=java.lang " , a String method parameter will be printed just as " String " and not as " java.lang.String " . Until now, only the packages for which the javadoc was printed were not fully qualified (which is the default behaviour of the regular javadoc). marcelschoen
Added correct handling of inner classes. They are now connected to their containing class in the bookmarks navigation frame. Fixes 821282. marcelschoen
Fixed brackets missing for array method arguments in method summary. Fixes 811384. marcelschoen
Fixed all parameters in summaries have the name of the first param. Fixes 833186. marcelschoen
Implemented the < EM > tag as a copy of the < I > tag (just italic text). marcelschoen
Also fixed the " broken " binary package of 0.5.1 (version number of JAR file was invalid in build.xml and shell script). This will not happen again as I've changed the build target which builds the binary package. It uses an ANT filter now to make sure the version number is always up-to-date. marcelschoen
